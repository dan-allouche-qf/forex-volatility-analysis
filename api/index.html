    Data & preprocessing

    fxvol.data

    Data acquisition and snapshot loading.

    By default the analysis loads a committed parquet snapshot so results are deterministic and network-free. refresh=True re-pulls live from Yahoo Finance.

    download_fx(tickers, start, end)

    Download daily OHLC for each pair from Yahoo Finance into a long DataFrame.

    Returns columns: Date, Open, High, Low, Close, pair. Only real trading days are returned (no calendar fabrication).

    load_ohlc(refresh=False)

    Load the long-format OHLC table (Date, OHLC, pair).

    Loads the committed snapshot unless refresh=True, in which case it re-downloads live and overwrites the snapshot.

    to_wide_close(long)

    Pivot the long table to a wide close-price matrix (index=Date, cols=pair).

    to_wide_ohlc(long)

    Pivot to a wide OHLC matrix with {pair}_{field} columns.

    fxvol.preprocessing

    Trading-day alignment.

    Critical design choice: FX trades Monday-Friday, so weekend dates are non- observations, not missing data. We therefore NEVER reindex onto a full calendar and we NEVER fabricate prices. Pairs are aligned on their actual shared trading dates; genuine single-day holiday gaps (a date present for some pairs but not others) are forward-filled at the price level only, with returns on such a day left to come out near zero rather than inventing a jump.

    align_close(long, how='union')

    Align pair close prices on real trading dates.

    Parameters

    long : long-format OHLC table (Date, OHLC, pair). how : "union" keeps every date traded by any pair and forward-fills the rare single-day holiday gaps (default; maximizes usable history). "inner" keeps only dates traded by all pairs (zero fill at all).

    Returns a wide close-price matrix on a pure trading-day index. Guarantees: no weekend rows, no NaNs after the first valid observation, no fabricated weekend values.

    trading_day_summary(wide)

    Quick integrity summary used in the notebook to prove no fabrication.

    Indicators & realized volatility

    fxvol.indicators

    Return, volatility and technical-indicator computations.

    All functions operate on a trading-day-indexed price/return frame. Annualization uses trading_days (== observations per year of the series), so the factor and the data frequency are always consistent.

    log_returns(prices)

    Log returns, ln(P_t / P_{t-1}). Time-additive; the convention used throughout for volatility and aggregation.

    simple_returns(prices)

    Arithmetic returns, P_t / P_{t-1} - 1 (used for P&L compounding).

    rolling_volatility(returns, window, trading_days=252)

    Annualized rolling volatility = rolling std(returns) * sqrt(trading_days).

    window is counted in trading days because returns is a trading-day series, so a window of 30 is genuinely 30 trading days.

    ewma_volatility(returns, lam=0.94, trading_days=252)

    Annualized RiskMetrics EWMA volatility.

    Variance recursion v_t = (1-lam) * r_t^2 + lam * v_{t-1} (adjust=False), annualized to a volatility.

    sma(prices, window)

    Simple moving average of prices over window trading days.

    rsi(prices, window=14, method='wilder')

    Relative Strength Index.

    method="wilder" uses Wilder's recursive smoothing (EMA with alpha = 1/window) - the canonical RSI. method="cutler" uses a simple moving average of gains/losses (Cutler's RSI). The two are different estimators; the choice is explicit rather than mislabeled.

    fxvol.realized

    Range-based realized-variance estimators (Parkinson, Garman-Klass, Rogers-Satchell).

    The committed snapshot carries full OHLC but the close-only pipeline ignores High/Low. Range estimators use the intraday range and are several times more efficient than the squared-return proxy. Caveat: they assume no overnight gap, no drift (Parkinson) and no jumps -- assumptions that bite in 24/5 FX daily bars, so results are reported alongside the r^2 proxy, never as a silent replacement.

    parkinson(high, low)

    Parkinson (1980) daily variance from the high-low range.

    garman_klass(open_, high, low, close)

    Garman-Klass (1980) daily variance from OHLC.

    rogers_satchell(open_, high, low, close)

    Rogers-Satchell (1991) daily variance (drift-robust).

    realized_variance(long_ohlc, pair, estimator='parkinson')

    Daily range-based variance (decimal return^2 units) for one pair.

    realized_proxy(long_ohlc, pair, estimator='parkinson', scale=100.0)

    Range-based realized variance rescaled to (scale*return)^2 units, so it is unit-compatible with :func:fxvol.models.walk_forward_variance forecasts (drop-in replacement for its realized column when scale=100).

    Models

    fxvol.models

    Conditional-volatility models and an out-of-sample forecast horse race.

    Baselines (EWMA, rolling, naive random-walk) vs GARCH(1,1) Student-t, scored on 1-day-ahead variance with QLIKE and MSE against the squared-return proxy, with a Diebold-Mariano test for significance.

    fit_garch(returns, dist='t', p=1, q=1, o=0, trading_days=252)

    Fit a GARCH(p,o,q) with the chosen innovation distribution (o>0 => GJR).

    standardized_residuals(fit)

    Standardized residuals z = resid/conditional_vol of a fitted GARCH (unit variance under correct specification); the input to EVT tail modelling.

    walk_forward_variance(returns, split=0.6, refit_every=5, ewma_lambda=0.94, roll_window=21, dist='t')

    One-day-ahead variance forecasts on an expanding window (daily units, % scale).

    Returns a DataFrame indexed by the forecast (t+1) date with columns: garch, ewma, rolling, naive (variance forecasts) and realized (the r_{t+1}^2 proxy). naive is the expanding unconditional (constant) variance. GARCH is refit every refit_every steps for speed.

    walk_forward_garch(returns, refit_every=10, dist='t', min_train=250, levels=(0.95, 0.99))

    Expanding-window 1-step-ahead GARCH(1,1) filter for conditional risk.

    For each out-of-sample date t+1 (using only information up to t) returns the conditional volatility forecast and the pieces needed for conditional VaR/ES:

    • sigma : 1-step conditional volatility, in DECIMAL return units
    • mu : fitted constant mean, decimal
    • nu : fitted Student-t degrees of freedom (NaN if dist!='t')
    • fhs_q_{L} : empirical lower-tail quantile (mass 1-L) of in-sample standardized residuals z = resid/sigma (for Filtered Historical Simulation)
    • fhs_es_{L} : -mean(z | z <= fhs_q_{L}) (positive ES factor)

    GARCH is refit every refit_every steps; the 1-step forecast is recomputed every day. No look-ahead: row at t+1 uses only returns up to and including t.

    forecast_losses(wf)

    Per-observation QLIKE and MSE loss series per model.

    Observations with a zero realized proxy (degenerate flat days) are dropped so QLIKE stays finite; the same mask is applied to every model for fair comparison and for the Diebold-Mariano test.

    score_forecasts(wf)

    Mean QLIKE and MSE per model (lower is better).

    diebold_mariano(loss_a, loss_b, h=1)

    Diebold-Mariano test on two loss series (HAC variance, h-1 lags).

    Negative statistic => model A has lower loss (better) than B.

    Risk

    fxvol.risk

    Risk metrics: VaR / Expected Shortfall, drawdown, Sharpe/Sortino, and VaR backtests (Kupiec POF + Christoffersen).

    Convention: returns are daily; VaR and ES are reported as POSITIVE loss numbers at a confidence level (e.g. 0.95). alpha = 1 - level is the tail mass.

    drawdown_curve(returns, log=False)

    Drawdown series (<= 0) from a return stream.

    max_drawdown(returns, log=False)

    Max drawdown with peak/trough dates and duration (in observations).

    kupiec_pof(n, exceptions, level=0.95)

    Kupiec unconditional-coverage (Proportion Of Failures) test.

    christoffersen(exceedances, level=0.95)

    Christoffersen independence + conditional-coverage tests on the 0/1 exception sequence.

    risk_table(returns, levels=(0.95, 0.99), trading_days=252)

    Per-pair risk summary used as a headline deliverable.

    pinball_loss(returns, quantile_pred, alpha)

    Quantile (pinball/tick) loss at level alpha for a predicted return quantile (quantile_pred = -VaR). Lower is better; proper for VaR.

    rolling_var_backtest(returns, garch_wf, levels=(0.95, 0.99), window=250)

    Backtest UNCONDITIONAL (rolling historical, rolling Student-t) vs CONDITIONAL (GARCH-filtered Student-t, Filtered Historical Simulation) 1-day VaR on the same out-of-sample dates, scored by breaches, Kupiec, Christoffersen and pinball loss.

    garch_wf is the output of :func:fxvol.models.walk_forward_garch.

    acerbi_szekely_z2(returns, var, es, level)

    Acerbi-Szekely (2014) Test 2 statistic. E[Z2]=0 under correct ES; Z2 < 0 => ES underestimates tail risk.

    es_backtest_garch_t(returns, garch_wf, level=0.99, n_sim=2000, seed=0)

    Acerbi-Szekely Z2 ES backtest for the GARCH-filtered Student-t model, with a simulated p-value from the model's own predictive distribution (left tail).

    vol_model_var_scorecard(returns, wf_var, levels=(0.95, 0.99), nu=None)

    Score each volatility model in a :func:fxvol.models.walk_forward_variance frame by the quality of the 1-day VaR it implies (breaches vs target, Christoffersen independence, pinball loss). Every model gets the SAME distributional assumption (Normal if nu is None, else standardized Student-t), so the comparison isolates the volatility forecast itself.

    Extreme value theory

    fxvol.evt

    Extreme Value Theory tails: Peaks-Over-Threshold (Generalized Pareto) on GARCH-standardized residuals (McNeil-Frey: GARCH for the body dynamics, EVT for the tail). A single fitted Student-t imposes a symmetric polynomial tail; EVT lets the data choose the tail index.

    gpd_fit(losses, threshold_quantile=0.9)

    Fit a Generalized Pareto Distribution to exceedances of losses over a high threshold (loc fixed at 0). losses are positive tail values (e.g. -standardized_residual).

    pot_var_es(losses, level=0.99, threshold_quantile=0.9)

    POT VaR/ES at confidence level on the scale of losses (standardized).

    Uses the GPD tail estimator F-bar(x) = (Nu/n)(1 + shape*(x-u)/scale)^(-1/shape).

    mean_excess(losses, n_points=40)

    Mean-excess function e(u) = E[X-u | X>u] over a grid of thresholds, used to justify the POT threshold (linear-in-u region indicates GPD applicability).

    conditional_pot_var_es(sigma_next, std_resid_losses, level=0.99, threshold_quantile=0.9)

    McNeil-Frey conditional VaR/ES: POT tail of standardized-residual losses, rescaled by the 1-step conditional volatility sigma_next.

    Backtesting & statistics

    fxvol.backtest

    Vectorized SMA-crossover backtest with honest accounting.

    No look-ahead: the position held on day t is decided from the signal observed at the close of t-1 (signal.shift(1)), i.e. next-bar execution. Transaction costs are charged on turnover. A buy-and-hold benchmark is always returned for comparison.

    sma_crossover_signal(prices, fast=20, slow=50)

    Long/flat target position (1/0) from an SMA crossover, decided at each close.

    tsmom_signal(prices, lookback=60)

    Time-series-momentum signal: +1/-1 = sign of the trailing lookback-day log return (Moskowitz-Ooi-Pedersen). Long/short, decided at each close.

    vol_target_position(signal, forecast_vol, target_vol=0.1, max_leverage=3.0)

    Scale a signal to a constant annualized volatility target using a (lagged-safe) conditional-vol forecast: pos = signal * target/forecast, capped at max_leverage. forecast_vol must use only info up to t.

    backtest(prices, signal, cost_bps=1.0, trading_days=252)

    Backtest a long/flat signal on prices with per-turnover costs.

    fxvol.stats_backtest

    Statistical significance of backtests.

    Turns point-estimate Sharpe ratios into inferential claims: a stationary block bootstrap (Politis-Romano) for confidence intervals, the Probabilistic Sharpe Ratio (Bailey & Lopez de Prado) accounting for skew/kurtosis and sample size, and the Deflated Sharpe Ratio that discounts for the number of configurations tried.

    stationary_bootstrap_indices(n, expected_block, n_boot, rng)

    Politis-Romano stationary bootstrap row indices, shape (n_boot, n).

    Block lengths are geometric with mean expected_block (probability p = 1/expected_block of starting a fresh block each step), preserving serial dependence in the resampled series.

    sharpe_bootstrap(returns, trading_days=252, expected_block=10.0, n_boot=2000, seed=0)

    Bootstrap distribution of the annualized Sharpe ratio.

    Returns the point estimate, a 95% percentile CI, and the bootstrap probability that the true Sharpe is <= 0.

    probabilistic_sharpe_ratio(returns, sr_benchmark=0.0)

    PSR: probability the true (per-period) Sharpe exceeds sr_benchmark, adjusting for non-normality (skew, kurtosis) and sample size.

    expected_max_sharpe(var_trials_sr, n_trials)

    Expected maximum (per-period) Sharpe across n_trials independent trials under the null of zero true Sharpe (Bailey & Lopez de Prado).

    deflated_sharpe_ratio(returns, n_trials, var_trials_sr)

    DSR: PSR against the expected-maximum-Sharpe benchmark for n_trials configurations, i.e. the probability the result survives selection bias. Requires n_trials >= 2.

    Diagnostics

    fxvol.diagnostics

    Econometric diagnostics: normality, stationarity, autocorrelation and ARCH effects. Claims of "fat tails" / "volatility clustering" in the write-up are substantiated by these tests rather than asserted.

    diagnostics_table(returns, prices=None, lags=10)

    Per-pair diagnostics table.

    Columns: skew, excess kurtosis, Jarque-Bera (p), ADF on returns (p), KPSS on returns (p), Ljung-Box on returns and on squared returns (p), and Engle ARCH-LM (p). Low normality p-value => reject normality (fat tails); low ARCH-LM / squared-return Ljung-Box p-value => volatility clustering.
